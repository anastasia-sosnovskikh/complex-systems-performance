Define log-posterior function

Define the log-posterior function lm_logpost(). Use the log-transformation on \(\sigma\), in other words, define the log-posterior in terms of the mean trend \(\boldsymbol{\beta}\)-parameters and the unbounded noise parameter, \(\varphi = \log\left[\sigma\right]\).

lm_logpost <- function(unknowns, my_info)
{
  # specify the number of unknown beta parameters
  length_beta <- my_info$design_matrix %>% ncol() 
  
  # extract the beta parameters from the `unknowns` vector
  beta_v <- unknowns[1:length_beta]
  
  # extract the unbounded noise parameter, varphi
  lik_varphi <- unknowns[length_beta + 1]
  
  # back-transform from varphi to sigma
  lik_sigma <- exp(lik_varphi)
  
  # extract design matrix
  X <- my_info$design_matrix
  
  # calculate the linear predictor
  mu <- as.vector(X %*% as.matrix(beta_v))
  
  # evaluate the log-likelihood
  log_lik <- sum(dnorm(x = my_info$yobs,
                       mean = mu,
                       sd = lik_sigma,
                       log = TRUE))
  
  # evaluate the log-prior
  log_prior_beta <- sum(dnorm(x = beta_v,
                              mean = my_info$mu_beta,
                              sd = my_info$tau_beta,
                              log = TRUE))
  
  log_prior_sigma <- dexp(x = lik_sigma, 
                          rate = my_info$sigma_rate, 
                          log=TRUE)
  
  # add the mean trend prior and noise prior together
  log_prior <- log_prior_beta + log_prior_sigma
  
  # account for the transformation
  log_derive_adjust <- lik_varphi
  
  # sum together
  log_lik + log_prior + log_derive_adjust
}

Use Laplace Approximations estimate to the Evidence.

From the lectures, we know that \(\log \left[ \mathrm{Evidence} \right]\) is equal to the following:

\[ \log \left[ \mathrm{Evidence} \right] \propto \frac{P}{2} \log \left( 2 \pi \right) - \frac{1}{2} \log \left( \left| \mathbf{H} \right|_{\hat{\theta}} \right) + \log \left( p \left( \mathbf{y} | \mathbf{\Phi}, \mathbf{\hat{\theta}} \right) \right) + \log \left( p \left( \mathbf{\hat{\theta}}\right) \right) \]

where \(P\) is the number of unknown parameters, \(\mathbf{\hat{\theta}}\) is the posterior mode of all \(P\) parameters, and \(\left| \mathbf{H} \right|_{\hat{\theta}}\) is the determinant of the Hessian matrix evaluated at the posterior mode. Thus, we can finish the function as follows:

my_laplace <- function(start_guess, logpost_func, ...)
{
  # code adapted from the `Learn Bayes`` function `laplace()`
  fit <- optim(start_guess,
               logpost_func,
               gr = NULL,
               ...,
               method = "BFGS",
               hessian = TRUE,
               control = list(fnscale = -1, maxit = 1001))
  
  mode <- fit$par
  post_var_matrix <- -solve(fit$hessian)
  p <- length(mode)
  int <- p/2 * log(2 * pi) + 0.5 * log(det(post_var_matrix)) + logpost_func(mode, ...)
  # package all of the results into a list
  list(mode = mode,
       var_matrix = post_var_matrix,
       log_evidence = int,
       converge = ifelse(fit$convergence == 0,
                         "YES", 
                         "NO"),
       iter_counts = as.numeric(fit$counts[1]))
}

Evidence-based assessment

The Evidence or marginal likelihood is attempting to evaluate generalization by integrating the likelihood over all a-priori allowed parameter combinations. If the Evidence can be calculated, it can be used to weight all models relative to each other. Thereby allowing you to assess which model appears to be most probable.

get_post_weights <- function(mod_list){
  model_log_evidence <- purrr::map_dbl(mod_list, 'log_evidence')
  evidence <- model_log_evidence %>%  exp()
  total_evidence <- evidence %>% sum()
  model_weights <- evidence / total_evidence
  model_weights
}
