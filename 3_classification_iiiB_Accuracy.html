get_CATEGORICAL_features_list <- function(df){
  colnames(df)[grepl('factor|logical|character',sapply(df,class))]

Tune the Gradient Booster Tree model.

  • nrounds controls the maximum number of iterations. For classification, it is similar to the number of trees to grow.

  • eta controls the learning rate, i.e., the rate at which our model learns patterns in data. After every round, it shrinks the feature weights to reach the best optimum. Lower eta leads to slower computation. It must be supported by increase in nrounds.

  • max_depth controls the depth of the tree.

  • gamma controls regularization (or prevents overfitting). Higher the value, higher the regularization. Regularization means penalizing large coefficients which don’t improve the model’s performance. The default is 0 which means no regularization.

  • colsample_bytree controls the number of features (variables) supplied to a tree. Typically, its values lie between (0.5,0.9).

  • min_child_weight, in regression, it refers to the minimum number of instances required in a child node. In classification, if the leaf node has a minimum sum of instance weight (calculated by second order partial derivative) lower than min_child_weight, the tree splitting stops. In other words, it blocks the potential feature interactions to prevent overfitting.

  • subsample controls the number of samples (observations) supplied to a tree. Typically, its values lie between (0.5-0.8).
