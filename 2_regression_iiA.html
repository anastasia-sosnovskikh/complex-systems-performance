Plot the r-squared and the RMSE

Since we are doing regression, the valid regression evaluation metrics are: R-Squared, MAE, and Root Mean Squared Error (RMSE).

I dont want to use MAE because I want my performance metric to be consistent with how the coefficients are estimated in the first place.

Since the coefficients are estimated by minimizing the error, I am left with RMSE and r-squared as performance evaluation models for the regression problem I am trying to solve.

Note: R-Squared (ranges from 0 to 1) because its the worst one from the list. It doesnt measure goodness of fit very well. It can be arbitrarily low when the model is completely correct and it can be arbitrarily close to 1 when the model is wrong (sources: HOML book, https://data.library.virginia.edu/is-r-squared-useless/, and https://www.rpubs.com/frycast/rsq).

all_model_metrics %>% 
  dplyr::select(model_name, r.squared, sigma) %>% 
  tidyr::gather(key = 'key', value = 'value', -model_name) %>% 
  ggplot(mapping = aes(x = model_name, y = value)) +
  geom_line(size = 1.5,
            mapping = aes(group = key)) +
  geom_point(size = 4.5) +
  facet_wrap(~key, scales = 'free_y') +
  scale_x_discrete(guide = guide_axis(angle = 45)) +
  theme_bw()

Based on the graphs, mod_basis - a basis function I came up with based on my analysis of EDA - seems to perform the best because this model has the highest r-squared and lowest sigma.

Plot AIC and BIC

The Bayesian Information Criterion (BIC) penalizes the data fit, via the log-likelihood, based on the number of parameters. Thus, complex models with many parameters get penalized. Another metric called Akaike Information Criterion (AIC) does not penalize as strong as BIC but serves the same purpose. They both are better than r-squared and RMSE because they penalize complex models which may guard against overfitting.

all_model_metrics %>% 
  dplyr::select(model_name, AIC, BIC) %>% 
  tidyr::gather(key = 'key', value = 'value', - model_name) %>% 
  ggplot(mapping = aes(x = model_name, y = value)) +
  geom_line(size = 1.5,
            mapping = aes(group = key)) +
  geom_point(size = 4.5) +
  facet_wrap(~ key, scales = 'free_y') +

  theme_bw()

Based on the graphs, mod_basis - a basis function I came up with based on my analysis of EDA - seems to perform the best again because this model has the lowest AIC and BIC.
